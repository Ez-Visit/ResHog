# 安全政策

## 支持的版本

ResHog 是一个持续发布的开源项目，我们仅为最新版本提供安全更新。

| 版本 | 支持状态 |
|------|---------|
| 最新 Release | ✅ 安全更新 |
| 旧版本 | ❌ 不支持 |
| main 分支 | ⚠️ 开发中，不稳定 |

## 报告漏洞

我们非常重视 ResHog 的安全问题。如果你发现安全漏洞，请**不要**通过公开 Issue 提交，而是：

1. **发送邮件**至：**security@reshog.dev**
2. **邮件主题**以 `[SECURITY]` 开头
3. **包含以下信息**：
   - 漏洞的详细描述
   - 复现步骤
   - 影响范围评估
   - 如果有，建议的修复方案

### 响应时间

| 阶段 | 预期时间 |
|------|---------|
| 确认收到报告 | 48 小时内 |
| 初步评估反馈 | 5 个工作日内 |
| 修复发布 | 视严重程度，7-30 天内 |

### 披露政策

- 我们遵循**负责任披露**原则
- 修复发布后，我们会在 GitHub Advisory Database 公开漏洞详情
- 报告者的贡献将在致谢中列出（除非你选择匿名）

## 安全设计说明

ResHog 的安全设计要点：

| 设计点 | 措施 |
|--------|------|
| 数据存储 | 所有数据仅存储在本地 SQLite，不外传 |
| 网络通信 | HTTP API 仅监听 `localhost`，不接受外部连接 |
| 权限最小化 | 服务以 LocalSystem 运行（采集需要），UI 客户端以普通用户权限运行 |
| 无遥测 | ResHog 不收集任何使用数据，不上报任何信息 |
| 依赖审计 | CI 流水线集成依赖漏洞扫描 |

## 已知安全考量

- **服务以高权限运行**：采集所有进程的 CPU/内存/IO 数据需要 LocalSystem 权限。这是 PDH 性能计数器的限制。如需降权，将无法读取系统进程数据。
- **SQLite 文件权限**：数据库文件默认存储在 `C:\ProgramData\ResHog\`，权限继承自父目录。高安全环境应手动限制访问权限。
